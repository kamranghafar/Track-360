⚠️  CRITICAL INFORMATION ABOUT PYTHON BYTECODE COMPATIBILITY

If you're seeing errors like:
  "ImportError: bad magic number in 'dashboard_project'"

This means the bytecode was compiled with a different Python version than
the one you're using. READ THIS FILE FOR THE SOLUTION!

================================================================================
                          THE PROBLEM
================================================================================

PYTHON BYTECODE IS NOT CROSS-VERSION COMPATIBLE!

Each Python version (3.8, 3.9, 3.10, 3.11, 3.12, 3.13, 3.14, etc.) has a
different "magic number" in its compiled bytecode (.pyc) files.

Example:
--------
• Compiled with Python 3.13: Magic number b'\xf3\r\r\n'
• Running with Python 3.14:  Magic number b'\xf4\r\r\n'
• Result: ImportError - bytecode incompatible!

This is a fundamental Python limitation, not a bug in the software.

================================================================================
                      DETAILED ERROR EXPLANATION
================================================================================

ERROR MESSAGE:
--------------
ImportError: bad magic number in 'dashboard_project': b'\xf3\r\r\n'

WHAT IT MEANS:
--------------
The bytecode file has magic number b'\xf3\r\r\n' (Python 3.13)
But you're running a different Python version (e.g., 3.14)
Python versions have different magic numbers and can't read each other's bytecode

MAGIC NUMBER TABLE:
-------------------
Python 3.8:  b'\x55\x0d\r\n'
Python 3.9:  b'\x61\x0d\r\n'
Python 3.10: b'\x6f\x0d\r\n'
Python 3.11: b'\xa7\x0d\r\n'
Python 3.12: b'\xcb\x0d\r\n'
Python 3.13: b'\xf3\r\r\n'
Python 3.14: (Different - future version)

WHY THIS HAPPENS:
-----------------
Bytecode was pre-compiled with one Python version
You're trying to run it with a different Python version
Python checks magic number first and rejects mismatches

THE FIX:
--------
Run our setup script which compiles for YOUR Python version!

================================================================================
                      FREQUENTLY ASKED QUESTIONS
================================================================================

Q: Why not just include pre-compiled bytecode?
A: Python bytecode is version-specific and won't work across Python versions.

Q: Why not use PyInstaller instead?
A: PyInstaller creates large executables (~50-100 MB) and requires complex
   setup. Our approach is lightweight and flexible.

Q: Can I use Python 3.14 if compiled with 3.13?
A: NO! You must run setup on the target system to recompile for 3.14.

Q: Do I need to run setup on every system?
A: YES! Each system needs setup to compile for its specific Python version.

Q: What if I have the same Python version?
A: Even with the same version (e.g., 3.13.0 vs 3.13.1), it's safest to
   recompile on the target system to avoid any subtle differences.

Q: Can I skip compilation?
A: If you keep source files (answer NO to remove), Python can use the .py
   files directly. But for protection, compile and remove source.

Q: How long does setup take?
A: Usually 2-5 minutes: dependency installation (slowest), compilation (fast),
   optional source removal (instant).

Q: Can I automate setup?
A: Yes! You can pre-answer the prompts:
   echo yes | python setup.py
   (But you lose the interactive options)

Q: Is the bytecode secure?
A: Basic protection. Can be decompiled with effort. For enterprise security,
   use PyArmor or similar commercial tools.

================================================================================
                          TROUBLESHOOTING
================================================================================

PROBLEM: Setup fails with "Python not found"
SOLUTION: Install Python 3.8+ from python.org

PROBLEM: "Permission denied" during setup
SOLUTION: Run as administrator (Windows) or use sudo (Linux/Mac)

PROBLEM: Dependencies fail to install
SOLUTION: Check internet connection, try manual install:
          pip install -r requirements.txt

PROBLEM: Compilation fails
SOLUTION: Ensure .py files are present, check file permissions

PROBLEM: Still getting magic number error after setup
SOLUTION: Delete all .pyc files and .venv folder, run setup again

PROBLEM: Want to recompile for different Python version
SOLUTION: Delete .pyc files and .venv, use different Python to run setup
