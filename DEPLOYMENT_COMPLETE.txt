                             DEPLOYMENT STEPS
================================================================================

TO DEPLOY TO ANOTHER SYSTEM:
-----------------------------

1. COPY THIS ENTIRE FOLDER to the new system
   - Use USB drive, network share, cloud storage, etc.
   - Make sure to copy ALL files and folders
   - Especially important: db.sqlite3 (your database!)

2. VERIFY PYTHON is installed on target system
   - Required: Python 3.8 or higher
   - Check: python --version or python3 --version

3. RUN SETUP SCRIPT (first time only):
   Windows: Double-click setup.bat
   Linux/Mac: bash setup.sh
   Any platform: python setup.py

4. RUN STARTUP SCRIPT (every time you start):
   Windows: Double-click start_dashboard.bat
   Linux/Mac: bash start_dashboard.sh
   Any platform: python start_dashboard.py

5. ACCESS DASHBOARD in web browser:
   URL: http://127.0.0.1:8000 (or your chosen IP:port)


================================================================================
                          WHAT HAPPENS AT STARTUP
================================================================================

When you run the startup script:

1. Script checks if database exists
2. Prompts you for IP address:
   → Default: 0.0.0.0 (allows network access)
   → Alternative: 127.0.0.1 (local only)
   → Just press ENTER for default

3. Prompts you for port number:
   → Default: 8000
   → Choose different if 8000 is busy
   → Just press ENTER for default

4. Django server starts
5. Dashboard is accessible at the displayed URL

BACKUP YOUR DATA:
-----------------
The db.sqlite3 file contains all your data. Make regular backups!

SECURITY:
---------
Current configuration is for DEVELOPMENT. For production:
- Change SECRET_KEY in settings.py
- Set DEBUG = False in settings.py
- Configure proper ALLOWED_HOSTS
- Set up HTTPS/SSL

NETWORK ACCESS:
---------------
To access from other computers:
- Start with IP 0.0.0.0
- Find your IP: ipconfig (Windows) or ifconfig (Linux/Mac)
- Access via: http://YOUR_IP:PORT
- Ensure firewall allows the port
